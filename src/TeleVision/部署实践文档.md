# TeleVision 部署实践文档 — Quest 3 手势识别 + FPV 图传

## 模块职责

TeleVision 在 Fast-Drone-250 中负责：

- **手势识别**: Quest 3 WebXR 手部追踪 → 无人机控制指令
- **FPV 图传**: 摄像头/仿真画面 → 立体传输到 Quest 3 眼镜

---

## 原理简述

```
┌──────────────┐         HTTPS (localtunnel)         ┌────────────────────────┐
│  Quest 3     │ ◄─────────────────────────────────► │  Linux 服务器            │
│  浏览器 WebXR │                                      │                        │
│              │  WebSocket: 手部关键点 + 头部矩阵 →   │  vuer server :8012     │
│              │                                      │  TeleVision.py         │
│              │  ← 立体画面 (共享内存, JPEG 编码)     │                        │
│              │                                      │  SharedMemory          │
│              │                                      │  ├── 图像 720×2560×    │
│              │                                      │  ├── 左手 4×4          │
│              │                                      │  ├── 右手 4×4          │
│              │                                      │  ├── 关键点 25×3×2     │
│              │                                      │  └── 头部 4×4          │
│              │                                      │                        │
│              │                                      │  Preprocessor.py       │
│              │                                      │  坐标变换 → 无人机指令   │
└──────────────┘                                      └────────────────────────┘
```

**数据环路:**

1. Quest 3 浏览器打开 HTTPS 页面，加载 WebXR 场景
2. vuer 建立 WebSocket 连接，注册 HAND_MOVE 和 CAMERA_MOVE 回调
3. 摄像头画面写入共享内存（`multiprocessing.shared_memory`，零拷贝）
4. vuer 主循环读取共享内存 → JPEG 编码 → WebSocket 推送左右眼画面
5. Quest 3 手部/头部追踪数据通过 WebSocket 回传
6. Preprocessor 做坐标变换 (WebXR y-up → ROS z-up)，映射为无人机控制指令

## 安装步骤

### 1. 创建 conda 环境

```bash
conda create -n tv310 python=3.10 -y
conda activate tv310
```

### 2. 安装依赖

```bash
# 核心 WebXR 框架 (预发布版)
pip install vuer==0.0.32rc7

# 计算机视觉 + 数学
pip install numpy opencv-python pytransform3d PyYAML

# Web 框架
pip install aiohttp aiohttp_cors

# 兼容性修复: vuer 0.0.32rc7 依赖旧版 params-proto
pip install params-proto==2.12.1
```

### 3. localtunnel (网络穿透)

```bash
# 无需提前安装，首次运行时选 y
npx localtunnel --port 8012
```

localtunnel 无需注册，替代 ngrok（ngrok 免费版需绑卡）。

## 源码修复

本项目修复了两处兼容性问题：

### 修复 1: 延迟 WebRTC 导入

`teleop/TeleVision.py` 第 8 行，删除顶层导入：
```python
from webrtc.zed_server import *   # 删除这行
```

在 `if stream_mode == "webrtc":` 分支内添加：
```python
import ssl
import logging
import aiohttp_cors
from aiohttp import web
from webrtc.zed_server import RTC, Args, on_shutdown, index, javascript
```

**原因**: image 模式不需要 aiortc/av，避免未安装时报错。

### 修复 2: 补充 Value 导入

`teleop/TeleVision.py` 第 5 行，改为：
```python
from multiprocessing import Array, Process, shared_memory, Queue, Manager, Event, Semaphore, Value
```

**原因**: `self.aspect_shared = Value('d', 1.0)` 使用了 `Value` 但源码没导入。

## stream_camera.py 关键逻辑

```python
# 1. 创建共享内存 (720×2560×3 = 约 5.3MB, 零拷贝)
img_shape = (720, 2560, 3)
shm = shared_memory.SharedMemory(create=True, size=np.prod(img_shape))
img_array = np.ndarray(img_shape, dtype=np.uint8, buffer=shm.buf)

# 2. OpenTeleVision 启动 vuer 子进程, 监听 8012 端口
tv = OpenTeleVision((720, 1280), shm.name, Queue(), Event(), ngrok=True)

# 3. 主循环: 读摄像头 → 水平拼接 → 写入共享内存
while True:
    if camera_ok:
        _, frame = cap.read()                    # BGR
    else:
        frame = make_checkerboard(720, 1280)     # 降级画面

    stereo = np.hstack([frame, frame])           # 左右眼同画面
    np.copyto(img_array, stereo[..., ::-1])      # BGR→RGB, 零拷贝写入
    time.sleep(0.01)
```

## 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| vuer 导入 TypeError | Python 3.8 不支持 `tuple[Any,bool]` | 升级到 Python 3.10 |
| vuer `Proto` 导入失败 | 新版 params-proto API 不兼容 | `params-proto==2.12.1` |
| `from vuer import Vuer` 失败 | vuer 默认不含 aiohttp | `pip install 'vuer[all]'` |
| `Value` not defined | 源码缺少 import | multiprocessing 加上 Value |
| aiortc 模块缺失 | WebRTC 未安装 | 改为按需导入 |
| ngrok 要求绑卡 | ngrok 政策变更 | 换 localtunnel |
| `pip install av` 编译失败 | 系统缺 libavfilter-dev | image 模式无需 av |
| 503 tunnel unavailable | 服务未启动 | 先启动 stream_camera.py |
| 虚拟机摄像头无法读帧 | VM USB isochronous 限制 | 宿主机跑脚本 / 棋盘格降级 |
| Quest 进入 VR 后黑屏 | 图像未推到共享内存 | 检查终端报错, 刷新页面 |
